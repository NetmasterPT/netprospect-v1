# DEBUG-FOUND — incidentes encontrados pelo monitor de saúde

Índice dos problemas que o monitor (loop de 15 min) detetou. Cada linha aponta para um relatório em
`docs/incidents/`. Um incidente **OPEN** que continue a acontecer é **expandido** no mesmo relatório
(nova observação datada), não duplicado. Marca **CLOSED** quando deixar de ocorrer / for corrigido.

O monitor **não corrige código** — só lê documentação e o(s) ficheiro(s) de código onde o erro
ocorre, para dar contexto ao relatório.
